## Last Session Changes (S31)

- `pi4/hardware/audio_io.py`: `_find_mic_device_index()` matches `'capture'` (not `'array'`). `capture` = `plug→dsnoop→hw:wm8960` — handles 16000 Hz via plug rate conversion. `array` = raw dsnoop, no rate conversion, crashes at 16000 Hz.
- `pi4/core/config.py`: `CHANNELS = 1` → `CHANNELS = 2`. wm8960 is stereo hardware; forcing mono through plug produced silent/malformed audio to OWW.
- `pi4/assistant.py`: `mic_idx = _find_mic_device_index()` + `input_device_index=mic_idx` wired into `pa.open()`. Previously the detected index was computed but never passed.
- `pi4/services/wakeword.py`: Wyoming OWW `Detection` events carry no score field — `data.get("score")` returned `None` → `0.0` default → `0.0 < 0.9` rejected all detections. Fixed: default to `1.0` when no score present (OWW only fires Detection when already above threshold).
- Commit: 09977e4
