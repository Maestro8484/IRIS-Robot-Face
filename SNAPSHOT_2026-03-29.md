## 8. SERIAL PROTOCOL

**Pi4 → Teensy:**
```
EMOTION:NEUTRAL/HAPPY/CURIOUS/ANGRY/SLEEPY/SURPRISED/SAD/CONFUSED
EYES:SLEEP     -- sleep renderer starts, mouth snore animation
EYES:WAKE      -- restore eye, apply NEUTRAL, restore mouth intensity
EYE:n          -- switch default eye to index n (0–6)
MOUTH:x        -- set mouth expression (0=NEUTRAL ... 8=SLEEP/OFF)
```

**Teensy → Pi4:**
```
FACE:1    -- face detected (30s cooldown)
FACE:0    -- face lost
```

**Note:** The `[SR] frame=N` heartbeat was removed from sleep_renderer.h. Teensy no longer emits periodic sleep frame confirmations.

**Rule:** Only `assistant.py` TeensyBridge owns `/dev/ttyACM0`. Everything else uses UDP → `127.0.0.1:10500`.

## 9. SLEEP STATE MACHINE

```
EYES:SLEEP received (processSerial):
  eyesSleeping = true
  angryEyeActive = confusedEyeActive = false
  *** NO updateChangedAreasOnly(false) — REMOVED (was causing Teensy LPSPI hang) ***
  blankDisplays()           ← fillScreen(BLACK) + updateScreen() Path B (dirty-rect), drains SPI
  mouthSetSleepIntensity()  ← 0x01 (very dim, 3/32 duty cycle)
  sleepRendererInit()       ← reset ring/star/frame state
  "[DBG] EYES:SLEEP -- displays blanked"
  "[DBG] EYES:SLEEP -- starfield starting"

loop() while sleeping:
  processSerial()           ← always runs (commands still received)
  personSensor.read() SKIPPED   ← guarded with !eyesSleeping (I2C off during heavy SPI)
  renderSleepFrame()        ← throttled SR_FRAME_MS=150ms; uses Path B (dirty-rect)
                                draws starfield + shooting stars + moon + ZZZ
  mouthSleepFrame()         ← throttled 50ms (snore animation)
  return                    ← eye engine skipped

EYES:WAKE received:
  eyesSleeping = false
  displayLeft/Right → updateChangedAreasOnly(true)   ← already true, belt-and-suspenders
  mouthRestoreIntensity()   ← 0x05 (~33%)
  setEyeDefinition(saved)   ← restore eye
  applyEmotion(NEUTRAL)
  "[DBG] EYES:WAKE -- displays restored"
```

**Key insight — drawChar infinite recursion (ROOT CAUSE of sleep hang, NOW FIXED):**
`srDrawZzz()` calls `d->drawChar(cx, cy, 'Z', col, SR_BLACK, sz)` with 6 parameters.
GC9A01A_t3n.h had an inline 6-param wrapper: `drawChar(x,y,c,col,bg,size)` → called itself → stack overflow → Teensy hang.
Fixed by changing to `drawChar(x, y, c, color, bg, size, size)` (7-param overload).
Fix is in `.pio/libdeps/eyes/GC9A01A1_t3n/src/GC9A01A_t3n.h` line 443 AND auto-applied by `scripts/patch_gc9a01a.py`.

**Key insight — why updateChangedAreasOnly(false) was removed:**
`updateScreen()` in GC9A01A_t3n has two paths:
- Path A (`_standard=true && _updateChangedAreasOnly=false`): raw 57599-pixel dump via `writedata16_cont` — **hangs Teensy 4.0 LPSPI**
- Path B (`_updateChangedAreasOnly=true`): dirty-rect row-by-row write — works correctly

`fillScreen()` calls `updateChangedRange(0,0,w,h)` marking the full screen dirty, so Path B sends a complete frame regardless. No need for Path A.

**Key insight — EYES:SLEEP startup timing:**
Teensy requires ~15 seconds after reboot/flash before it reliably processes EYES:SLEEP via TeensyBridge.
The 9PM cron sends UDP immediately — if IRIS rebooted within the previous 15s, the command is lost.
Normal operation (no recent reboot) is unaffected.
