## 1. PROJECT OVERVIEW

Stationary interactive AI robot face. Two round TFT displays as eyes, pan servo for head tracking, voice pipeline via Pi4 Jarvis assistant. Wall-E aesthetic -- purple/pink acrylic eye housings, blue glitter acrylic base, laser-cut construction.

### Goals
- Eyes animate with realistic motion (TeensyEyes engine)
- Eyes express emotions driven by Jarvis AI responses
- Head pans to track faces via Person Sensor (servo system)
- Face detection triggers Jarvis greeting via Pi4
- Megan's complaint addressed: robot does NOT stare at people unless actively in a conversation

## 2. FINAL PHYSICAL ARCHITECTURE

### Base (fixed, stationary)
```
┌─────────────────────────────────────────────┐
│  Pi4                                        │
│   └── short 40-pin ribbon (~10cm)           │
│        └── ReSpeaker HAT                   │
│             └── mics exposed via front      │
│                  face cutout, forward-facing │
│                                             │
│  Pi camera -- fixed cutout front/top        │
│  Speaker   -- grille cutout front face      │
│  Eyes Teensy 4.0 -- USB to Pi4             │
│  Servo Teensy 4.0 -- USB power (standalone) │
│                                             │
│  Power in via USB through base              │
└─────────────────────────────────────────────┘
```

### Rotating Head (pan only, ±~45°)
```
┌─────────────────────────────────────────────┐
│  2x GC9A01A TFT displays (240x240 round)    │
│  Person Sensor -- center bridge (gaze)      │
│  Person Sensor -- center bridge (servo)     │
│  Wires down neck to base                    │
└─────────────────────────────────────────────┘
```

### Why This Layout
- ReSpeaker HAT has soldered mics -- cannot separate mics from board
- Short fixed 40-pin ribbon: no flex/fatigue, no rotation stress, ~10cm trivial signal integrity
- Pi4 protected inside base enclosure
- Mics face forward from fixed position -- better room pickup than rotating
- Camera fixed and forward-facing -- consistent framing for Jarvis vision queries
- Zero moving cables across the rotation joint (only display SPI + Person Sensor I2C cross the neck)
- Base volume is sufficient per physical inspection of photos

### What Crosses the Rotation Joint (neck wires)
- SPI lines to 2x TFT displays (Eyes Teensy in base → displays in head)
- I2C to Person Sensor for gaze (Eyes Teensy)
- I2C to Person Sensor for servo (Servo Teensy)
- Power (3.3V/5V + GND)
- All low-current signal lines -- manageable in a spiral wrap up the neck post

### Serial Protocol (USB, 115200 baud)
```

                EMOTION:HAPPY\n
                EMOTION:CURIOUS\n
                EMOTION:ANGRY\n
                EMOTION:SLEEPY\n
                EMOTION:SURPRISED\n
                EMOTION:SAD\n

Teensy → Pi4:   FACE:1\n   (new face detected, rate-limited 30s cooldown)
                FACE:0\n   (face lost)
```

| SLEEPY | 0.85 | yes | 5000 |
| SURPRISED | 0.95 | yes | 600 |
| SAD | 0.25 | yes | 4000 |

### Tracking Lockout Logic (Megan's fix)
- Boot state: `trackingActive = false`, `autoMove = true` -- eyes wander, ignore sensor
- Any non-NEUTRAL emotion from Pi4 → `trackingActive = true`, 15s window opens
- `EMOTION:NEUTRAL` → `trackingActive = false` immediately, autoMove resumes
- Window expiry (15s no new emotion) → tracking locks automatically
- FACE:1/0 always reported to Pi4 regardless of tracking state

### Timing Constants
```cpp
TRACKING_WINDOW_MS   = 15000   // ms after last emotion before tracking locks
FACE_LOST_TIMEOUT_MS =  5000   // ms no face → re-enable autoMove within session
FACE_COOLDOWN_MS     = 30000   // min ms between FACE:1 sends to Pi4
```


### Behavior
- Fully standalone -- no serial comms to Pi4 or Eyes Teensy
- Continuously reads Person Sensor via I2C
- Pans SG90 servo to track largest facing face X position
- Smoothed movement (exponential damping) to prevent jitter
- Idle behaviors:
  - No face detected: slow left-right sweep
  - Face lost mid-track: hold position 3s then resume sweep

### Hardware
- Teensy 4.0
- Person Sensor I2C address 0x62 on Wire (SDA/SCL)
- SG90 servo on pin 3 (PWM)

**Emotion sender:**
```python
def send_emotion(emotion):
    if teensy:
        try:
            teensy.write(f"EMOTION:{emotion}\n".encode())
        except Exception:
            pass

EMOTION_KEYWORDS = {
    'happy': 'HAPPY', 'excited': 'HAPPY', 'great': 'HAPPY',
    'curious': 'CURIOUS', 'interesting': 'CURIOUS', 'wonder': 'CURIOUS',
    'angry': 'ANGRY', 'frustrated': 'ANGRY',
    'sleepy': 'SLEEPY', 'tired': 'SLEEPY',
    'surprised': 'SURPRISED', 'wow': 'SURPRISED',
    'sad': 'SAD', 'sorry': 'SAD',
}

def send_emotion_from_text(text):
    for keyword, emotion in EMOTION_KEYWORDS.items():
        if keyword in text.lower():
            send_emotion(emotion)
            return
    send_emotion('NEUTRAL')
```

**Teensy listener thread (add near bottom of init, before main loop):**
```python
def teensy_listener():
    global last_face_time, in_conversation
    while True:
        try:
            if teensy and teensy.in_waiting:
                line = teensy.readline().decode().strip()
                if line == "FACE:1":
                    now = time.time()
                    if not in_conversation and (now - last_face_time) > FACE_COOLDOWN:
                        last_face_time = now
                        send_emotion("CURIOUS")
                        speak("Oh, hello. I see you.")
                elif line == "FACE:0":
                    send_emotion("NEUTRAL")
        except Exception:
            pass
        time.sleep(0.05)

threading.Thread(target=teensy_listener, daemon=True).start()
```
