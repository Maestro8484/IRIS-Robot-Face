**[FIX-1] `[SLEEP]` debug logging missing from person sensor block**

**[FIX-2] `mouthInit` brightness inconsistency**
- Status: Fixed
- Location: `mouth.h`, `mouthInit()`
- Before: `_writeAll(0x0A, 0x01)` (6.7%)
- After: `_writeAll(0x0A, 0x05)` (33%)
- Why: `mouthRestoreIntensity()` sets 0x05 as "normal" brightness. Starting at 0x01 meant mouth was dim on boot, then became brighter after the first sleep/wake cycle. Now startup matches post-wake brightness.

### NO ACTION NEEDED

**Serial buffer (32B) — SAFE**
- Longest command: `EMOTION:SURPRISED` = 17 chars. Buffer check `serialBufLen < SERIAL_BUF_SIZE - 1` correctly prevents overflow. `serialBufLen` resets on `\n`. Partial commands without `\n` persist but next command suffix won't parse correctly — acceptable edge case since Pi4 always terminates with `\n`.

**blankDisplays() sequence — CORRECT**
- `fillBlack()` waits for async DMA (`asyncUpdateActive()` with 200ms timeout), then calls `fillScreen(0x0000)` + `updateScreen()`. Both displays get a full black frame pushed to hardware synchronously before sleep renderer starts. No visual gap.

**Sleep guard logic — CORRECT**
- Person sensor tracking guarded by `if (!eyesSleeping)`. ✓
- Angry/Confused revert timers at lines 338-349 are AFTER `if (eyesSleeping) { return; }`. They only fire when awake. ✓
- EMOTION commands during sleep update eye state (pupil ratio, gaze ms) but don't render. `applyEmotion(NEUTRAL)` in EYES:WAKE handler clears angryEyeActive/confusedEyeActive. ✓
- EYE:n during sleep correctly updates `userDefaultEye` and `defIndex`. EYES:WAKE handler uses `defIndex = UINT32_MAX` force-reload trick to re-apply. ✓

**renderSleepFrame() — CORRECT**
- Both framebuffers fully built before any `updateScreen()` call. No partial tear. Left then right is sequential, but since each uses its own framebuffer, neither eye shows partial data.
- SR_FRAME_MS throttle (66ms ≈ 15fps) checked before any draw. `srLastFrameMs=0` in `sleepRendererInit()` guarantees first frame runs immediately.

**setTargetPosition() seeding fix (ed8fa41) — CORRECT**
- When tracking restarts (face re-detected), `state.eyeOldX/Y` is seeded from the current interpolated position via smoothstep. No snap-back to origin.

**EyeController::renderFrame() alternating eye — CORRECT**
- `eyeIndex` alternates 0/1 per call. Left eye rendered every other loop iteration. This is the upstream engine design; sleep renderer bypasses it entirely via `return` before `eyes->renderFrame()`.

**mouthSleepFrame() — NO RATE LIMIT (acceptable)**
- Called every loop iteration during sleep. MAX7219 SPI writes at full loop rate. Not a bug but wastes ~50µs per iteration. Not worth fixing — current behavior is stable.
