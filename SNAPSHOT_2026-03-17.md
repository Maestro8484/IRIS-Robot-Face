## Current Firmware State
- SPI speed: 20MHz (reduced from 40MHz for USB stability)
- ASYNC: false on both displays
- Boot state: NEUTRAL -- eyes wander, Person Sensor ignored
- Any non-NEUTRAL emotion from Pi4 serial opens 15s tracking window
- EMOTION:NEUTRAL restores full auto behavior
- `faceTrackingLocked` guard prevents repeated setAutoMove(false) per frame (was causing USB crash)
- Serial protocol: Pi4 sends `EMOTION:X\n`, Teensy sends `FACE:1\n` / `FACE:0\n` (FACE:1 rate-limited 30s cooldown)

## config.h -- CURRENT STATE
Currently in **dual-eye comparison mode**: Left=nordicBlue, Right=hazel.
To switch to production (both nordicBlue): change eyeDefinitions line to `{nordicBlue::eye, nordicBlue::eye}` and remove hazel include.

```cpp
#pragma once

#include "eyes/eyes.h"

#include "eyes/240x240/nordicBlue.h"
#include "eyes/240x240/hazel.h"
#include "eyes/EyeController.h"

#define USE_GC9A01A

#ifdef USE_GC9A01A
#include "displays/GC9A01A_Display.h"
#endif

// Left = nordicBlue, Right = hazel (comparison mode)
// For production: {nordicBlue::eye, nordicBlue::eye} and remove hazel include
std::array<std::array<EyeDefinition, 2>, 1> eyeDefinitions{{
    {nordicBlue::eye, hazel::eye},
}};

#ifdef USE_GC9A01A
GC9A01A_Config eyeInfo[] = {
    {0,  2, 26, 27, 3, 0, true,  true, false}, // Left  -- nordicBlue
    {10, 9, 11, 13, -1, 0, false, true, false}, // Right -- hazel
};
#endif

constexpr uint32_t EYE_DURATION_MS{10'000};
constexpr uint32_t SPI_SPEED{20'000'000};

constexpr int8_t BLINK_PIN{-1};
constexpr int8_t JOYSTICK_X_PIN{-1};
constexpr int8_t JOYSTICK_Y_PIN{-1};
constexpr int8_t LIGHT_PIN{-1};
constexpr bool USE_PERSON_SENSOR{true};

EyeController<2, GC9A01A_Display> *eyes{};

void initEyes(bool autoMove, bool autoBlink, bool autoPupils) {
  auto &defs = eyeDefinitions.at(0);
  auto l = new GC9A01A_Display(eyeInfo[0], SPI_SPEED);
  auto r = new GC9A01A_Display(eyeInfo[1], SPI_SPEED);
  const DisplayDefinition<GC9A01A_Display> left{l, defs[0]};
  const DisplayDefinition<GC9A01A_Display> right{r, defs[1]};
  eyes = new EyeController<2, GC9A01A_Display>({left, right}, autoMove, autoBlink, autoPupils);
}
```

## Key Learnings This Session
- Dual-eye comparison: `{nordicBlue::eye, hazel::eye}` in eyeDefinitions -- BOTH includes required at top of config.h
- Iris size controlled by `radius` in config.eye -- hazel=60, match this
- Procedural iris generation never matches photographic quality -- always start from hazel iris and recolor
- `read_media_file` on PNGs burns enormous context (base64) -- avoid in new sessions, do image work in photo editor
- SPI 20MHz + ASYNC false = stable. Do not raise until physical build is complete.
- tablegen generates `polarDist_240_125_68_0` for custom iris radius -- this is fine, file size ~1085KB vs hazel 916KB, both fit in Teensy 4.0 2MB flash
