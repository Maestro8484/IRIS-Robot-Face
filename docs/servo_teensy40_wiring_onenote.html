IRIS BASE MOUNT CONTROLLER (TEENSY 4.0) — Pin & Wire Reference

Last updated: 2026-05-29 (S75 — servo constants, udev symlinks, GY-PAJ7620 replacement, all 8 gestures)  |  Board: Teensy 4.0  |  Port (Pi4): /dev/ttyIRIS_SERVO  |  Baud: 115200

Teensy 4.1 (display controller) owns /dev/ttyIRIS_EYES (serial 13625440). This board is /dev/ttyIRIS_SERVO (serial 12763490). Udev symlinks survive USB port swaps and reboots. Do NOT use /dev/ttyACM* numbers — they change when cables move.

4. GY-PAJ7620 Breakout — Pin to Wire

HW-004: HiLetgo PAJ7620U2 (original unit) DEAD — I2C ACK=NO, 0x73 absent from bus scan, reflow failed. Physically still seated in enclosure.
PENDING: GY-PAJ7620 replacement module on order. Seats identically — same address 0x73, same VIN/GND/SDA/SCL wiring, no firmware changes required. INT unconnected (polling mode).

Address 0x73. Keep clear line of sight to sensor window. GY-PAJ7620 module has onboard LDO — connect VIN pin to 3.3V (some modules label this VCC; confirm pinout on delivery).

Install checklist (future session):

  1. Remove dead HiLetgo PAJ7620U2. Seat GY-PAJ7620 identically.
  2. Power cycle. Open serial monitor at 115200 within 8s.
  3. Confirm DIAG: PAJ7620U2 0x73 ACK=YES and DIAG: PAJ7620U2 init=OK.
  4. Flash TS40-S1 firmware (see HANDOFF_CURRENT.md). Set GESTURE_SENSOR_REQUIRED=True in config.py and deploy to Pi4.

6. USB Serial Commands (Teensy 4.0 → Pi4)

All 8 gesture commands dispatched via hardware/base_mount_bridge.py. Default Pi4 actions are configurable per-gesture in iris_config.json (GESTURE_MAP) via the web UI Gestures tab.

GY-PAJ7620 Gesture (physical)Serial CommandDefault Pi4 Action
Swipe UPVOL+\nVolume up
Swipe DOWNVOL-\nVolume down
Swipe LEFTSTOP\nInterrupt current TTS playback
Swipe RIGHTSTOP\nInterrupt current TTS playback
Push toward sensorFORWARD\nLISTEN (activate listen mode)
Pull away from sensorBACKWARD\nSLEEP
Clockwise wrist rotationCW\nVOL+
Counter-clockwise rotationCCW\nVOL-

Pi4 serial: /dev/ttyIRIS_SERVO (udev symlink, hardware serial 12763490) at 115200 baud — USB CDC native (no bridge chip).
Pi4 handler: hardware/base_mount_bridge.py daemon thread started by assistant.py.
Teensy 4.1 display controller owns /dev/ttyIRIS_EYES (hardware serial 13625440) via hardware/teensy_bridge.py.
Valid configurable Pi4 actions: VOL+, VOL-, STOP, LISTEN, SLEEP, WAKE, MUTE, SKIP.

7. Servo Control Constants (pan_servo.h)

ConstantValueNotes

PAN_SPEED0.02Legacy delta scale (face pixel offset → pan degrees). Still active.
FACE_HOLD_MS2500 msHold position after face lost before returning
FACE_RETURN_MS6000 msTime after face lost before center-return begins

Tracking: EASE_LINEAR, startEaseTo(filteredPan, PAN_TRACK_SPEED) — no isMoving() guard, servo continuously chases face.
Idle return: EASE_CUBIC_IN_OUT, startEaseToD(desiredPan, 100) — isMoving() guard active.
Torque release: panServo.detach() when within 1° of center at idle. Re-attach before next move.
